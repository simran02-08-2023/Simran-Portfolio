Distributed Rate Limiter

Project Overview

Distributed Systems

About

The Distributed Rate Limiter is designed to protect APIs from excessive traffic by controlling the number of requests a client can make within a specified time period. The project implements two industry-standard rate limiting algorithms: Token Bucket for burst traffic handling and Sliding Window for strict request enforcement. Redis Lua scripts ensure atomic operations and prevent race conditions when multiple server instances process requests simultaneously.

Algorithms Used

Rate Limiting Strategies

Token Bucket

Allows burst traffic by storing tokens inside a bucket. Each request consumes one token, while tokens are refilled periodically. If no tokens remain, requests are rejected.

Sliding Window

Tracks requests over a rolling time window using Redis Sorted Sets. Unlike Token Bucket, it enforces strict request limits without allowing bursts.

Engineering Decisions

Design Choices

  • Implemented Token Bucket to allow controlled traffic bursts.
  • Implemented Sliding Window for strict request limiting.
  • Used Redis Lua scripts to guarantee atomic operations.
  • Designed to prevent race conditions across multiple server instances.
  • Adopted a fail-open strategy to prioritize availability if Redis becomes unavailable.
  • Validated functionality with JUnit 5 and Mockito tests.
